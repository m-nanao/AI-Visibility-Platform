# 01. 要件定義

## 1. プロダクト概要

**LLMO / AI Visibility Platform** は、ブランド・企業名が ChatGPT や Perplexity、Google AI Overview などの生成AI/AI検索サービス上で「どのように認知されやすいか」を可視化し、認知度向上のための改善提案を行うWebツールである。

LLMO（LLM Optimization）は、従来のSEO（検索エンジン最適化）に対して「生成AIにどう扱われるか」を最適化する新しい領域であり、本プロダクトはその可視化・診断部分を担う。

## 2. 重要な前提（スコープの境界）

- 本ツールは **特定LLMの学習内容を完全に再現するものではない**。
- Common Crawl のような公開Webアーカイブや、DataForSEOなどの検索・AI Overviewデータをもとに、「このブランドはWeb上でこう語られている＝AIにもこう認知されやすいはずだ」という **推定・シミュレーション** を行うツールである。
- そのため、実際のLLMの出力と本ツールの推定結果は完全一致しない前提であり、UI・ドキュメント上でも「推定」「傾向分析」であることを明示する。

## 3. 背景・課題

- 企業のブランド担当者・マーケター・SEO担当者は、検索エンジンでの順位は把握できても、ChatGPTなどの生成AIが自社をどう説明するか・引用するかを把握する手段を持たない。
- AI検索（AI Overview等）経由の流入が増える中、「AIにどう見えているか」を定点観測し改善する需要が高まっている。
- 一方でLLMの学習データやプロンプトへの応答は不透明であり、直接検証するのは難しい。公開Webデータから推定するアプローチが現実的な代替手段となる。

## 4. ユーザー像（想定ペルソナ）

- 事業会社のマーケティング / ブランド担当者（LLMOという概念に関心があるが専門知識は薄い）
- SEO/コンテンツ担当者（既存SEO業務の延長でLLMO対応を検討している）
- 代理店・コンサルタント（クライアント企業向けにAI認知度レポートを作成したい）

## 5. MVPスコープ（現状実装済み）

- Webアプリ（Next.js + TypeScript + Tailwind CSS）
- トップページにブランド名入力フォームを設置
- 「分析開始」ボタン押下で、ダミーデータに基づく分析結果を表示
- 表示内容（5セクション）
  1. ブランド認知サマリー
  2. 共起語ランキング
  3. 文脈分析
  4. AI Overview比較
  5. 改善提案
- `/app/api/analyze`（POST）に固定JSONを返すAPIルートを実装済み（フロントとはまだ未接続）
- 実データ取得・DB永続化は未実装（すべてダミーデータ・固定レスポンス）

## 6. MVP後のスコープ（将来拡張）

- Common Crawl からのWebデータ収集・共起語/文脈抽出
- DataForSEO 連携による検索結果・AI Overview掲載状況の取得
- Python製分析API（形態素解析・共起語抽出・センチメント分析・ランキング計算など）
- PostgreSQL による分析結果・ブランド・収集元データの永続化
- 分析履歴の閲覧、複数ブランドの比較

詳細は [02_roadmap.md](./02_roadmap.md) を参照。

## 7. 非機能要件

- 日本語UIを基本とする（英語ブランド名の入力にも対応）
- レスポンシブ対応（PC/タブレット想定、管理画面的なレイアウト）
- ライト/ダークモード両対応（Tailwindの `dark:` バリアントを使用）
- 将来的なAPI化・DB接続に備え、フロントのダミーデータはロジックから分離して管理する（`app/lib/dummy-data.ts`）
- Node.js 20.9 以降が必須（利用中のNext.jsバージョンの要件）

## 8. 非スコープ（MVP時点で対応しないこと）

- ユーザー認証・アカウント管理
- 課金・プラン管理
- 分析結果の永続保存・履歴管理
- 実データに基づく分析（Common Crawl / DataForSEO / Python分析APIとの結合）

##  9. 成功条件

このツールの目的はブランドがAIにどのように認知されやすいかを推定し、改善施策につなげられること。
以下を満たしたらMVP完成とする。

- ブランド分析
- 共起解析
- 文脈分析
- AI Overview比較
- 改善提案
