## 2. 分析画面（トップ、実装済み）

### 目的

ブランド名を入力し、AI上での認知度の推定結果を5つの観点で確認する、本プロダクトの中心画面。

### レイアウト

```
┌───────────────────────────────────────────────────────┐
│ LLMO / AI Visibility Platform                          │  ← ヘッダー
│ ブランドがAIサービス上でどのように認知されているかを分析します        │
├───────────────────────────────────────────────────────┤
│ ┌───────────────────────────────────────────────────┐ │
│ │ ブランド名 [____________________]      [分析開始]    │ │  ← 入力フォームカード
│ └───────────────────────────────────────────────────┘ │
│                                                         │
│ 「〇〇」の分析結果（ダミーデータ）              [リセット] │
│ ┌───────────────────────────────────────────────────┐ │
│ │ 1. ブランド認知サマリー（2カラム分の幅）               │ │
│ └───────────────────────────────────────────────────┘ │
│ ┌───────────────────────┐ ┌───────────────────────┐   │
│ │ 2. 共起語ランキング       │ │ 3. 文脈分析             │   │
│ └───────────────────────┘ └───────────────────────┘   │
│ ┌───────────────────────┐ ┌───────────────────────┐   │
│ │ 4. AI Overview比較      │ │ 5. 改善提案             │   │
│ └───────────────────────┘ └───────────────────────┘   │
└───────────────────────────────────────────────────────┘
```

実装対応: [app/page.tsx](../app/page.tsx)、[AnalysisDashboard.tsx](../app/components/AnalysisDashboard.tsx)、`app/components/sections/*`

### 状態遷移

```
idle（未入力/未実行）
  │ 「分析開始」押下（brandName入力済み）
  ▼
loading（分析中... を表示、ボタン無効化）
  │ 成功                          │ 失敗
  ▼                              ▼
done（5セクション表示 + リセット）   error（エラーメッセージ表示）
  │ 「リセット」押下
  ▼
idle
```

### 各セクションの表示要素

| セクション | 主な表示要素 |
| --- | --- |
| 1. ブランド認知サマリー | 認知スコア、総言及数、主要プラットフォーム、センチメント内訳（バー）、サマリー文章 |
| 2. 共起語ランキング | 順位、キーワード、出現回数、トレンド（↑↓→）、バー |
| 3. 文脈分析 | 文脈名、説明、センチメントバッジ、引用例 |
| 4. AI Overview比較 | プラットフォーム名、掲載有無、順位、概要（テーブル形式） |
| 5. 改善提案 | 施策タイトル、説明、優先度バッジ |

### 今後の変更予定

- Phase 2: フォーム送信先を `/api/analyze` に変更（表示・レイアウトは変更しない想定）
- Phase 5以降: 各セクションの項目に「情報源」バッジ・リンクを追加し、[04_data_model.md](./04_data_model.md) の `analysis_sources` を参照できるようにする（下記4章）

## 3. ブランド一覧画面（未実装）

### 目的

過去に分析したブランドを一覧し、再分析や履歴確認への入口とする。

### レイアウト（案）

```
┌───────────────────────────────────────────────────────┐
│ ブランド一覧                              [+ 新規ブランド] │
├───────────────────────────────────────────────────────┤
│ ブランド名     最終分析日        認知スコア    操作         │
│ OpenAI        2026-07-09        78          [再分析][履歴]│
│ Anthropic     2026-07-08        —           [再分析][履歴]│
└───────────────────────────────────────────────────────┘
```

- 「新規ブランド」押下でモーダル or 簡易フォームを表示し、`POST /api/brand` を呼ぶ。
- 「再分析」押下でトップ画面（分析画面）に遷移し、当該ブランド名を入力済みの状態で開始できるようにする。
- 「履歴」押下で分析履歴画面（4章）に遷移する。
- 使用API: `GET /api/brand`, `POST /api/brand`

## 4. 分析履歴画面（未実装）

### 目的

過去の分析結果を振り返り、必要であれば「この結果はどこから来たか」を確認できるようにする。

### レイアウト（案）— 一覧

```
┌───────────────────────────────────────────────────────┐
│ 分析履歴                        ブランド: [OpenAI ▾]      │
├───────────────────────────────────────────────────────┤
│ 実行日時            ステータス   認知スコア                │
│ 2026-07-09 10:00   完了        78            [詳細を見る] │
│ 2026-07-01 09:30   完了        72            [詳細を見る] │
└───────────────────────────────────────────────────────┘
```

### レイアウト（案）— 詳細（`/history/:analysisId`）

トップ画面の分析結果（2章のセクション1〜5）と同じダッシュボードを再利用して表示する。加えて、各項目の横に情報源バッジを表示する:

```
┌───────────────────────────────────────────────────────┐
│ 2. 共起語ランキング                                       │
│ 1. 料金プラン   42件  ↑   [情報源: PR TIMES, News]        │
│ 2. 導入事例     35件  ↑   [情報源: Common Crawl]          │
└───────────────────────────────────────────────────────┘
```

- 情報源バッジはクリックで参照元URL一覧を表示（ポップオーバー等）。
- 使用API: `GET /api/history`, `GET /api/history/:analysisId`
- データ根拠: [04_data_model.md](./04_data_model.md) の `analysis_sources` / `analysis_result_sources`

## 6. 共通ナビゲーション（画面が増えた際に追加）

現状（分析画面のみ）はナビゲーションなしのシングルページ構成。ブランド一覧・履歴・設定画面を追加するタイミングで、ヘッダーに簡易ナビゲーションを追加する。

```
┌───────────────────────────────────────────────────────┐
│ LLMO / AI Visibility Platform   分析｜ブランド｜履歴｜設定 │
└───────────────────────────────────────────────────────┘
```

- 現状のヘッダー（[app/page.tsx](../app/page.tsx)）にナビゲーションリンクを追加する形で対応し、レイアウトの大枠（ヘッダー＋`max-w-5xl`のメインエリア）は踏襲する。

## 7. デザインの共通方針

- Tailwind CSSによる管理画面風レイアウト（`zinc`系グレースケール＋アクセントカラーはセンチメント/トレンド/優先度バッジのみ）
- ライト/ダークモード両対応（`dark:` バリアント）
- カードベースのレイアウト（[Card.tsx](../app/components/Card.tsx) を共通コンポーネントとして再利用）
- 新規画面を追加する際も、既存の `Card` コンポーネントとバッジスタイル（[badge-styles.ts](../app/lib/badge-styles.ts)）を再利用し、見た目の一貫性を保つ
